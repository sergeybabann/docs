1 Настройка окружения

Требования для проекта

В корневом каталоге проекта лежит файл requirements.txt в котором прописаны необходимые требования для работы с проектом. Это будет приложение Single Page Application, работающее в основном на языке JavaScript.

Будем работать со следующими технологиями:

  • Webpack (Babel, SCSS, Eslint)
  • Существует 2 режима разработки:
    • dev mode (Dev Server, Source Maps, Eslint, Not Minified JS & CSS
    • prod mode (Minified Code)
  • GIT
  • Существует 2 версии приложения:
    • Production
    • Development
    Develoment - версия прокта где ведется разработка
    Production - версия, которую видит пользователь
    Подробное описание о версиях в video start: 9.47

Настройка и работа с GIT
  • Создаем в корневом каталоге проекта файл .gitignore Отмечаем в нем директории и файлы, которые не будут отслеживаться системой GIT. Отмеченные директории:
    • dist
    • node_modules
  • Связываем внешний и локальный репозитории по стандартной схеме представленной в github. Добавляем все файлы в отслеживание и делаем push на ветке main.
  • Создаем отдельную ветку для работы с Webpack. Ввводим в консоли команду: git checkout -b "webpack" и нажимаем клавишу enter, (в консоли должно появиться сообщение "Переключено на новую ветку «webpack»"), если оно появилось, то переключение прошло успешно, и мы переключились на ветку webpack. Делаем push на ветку webpack.
