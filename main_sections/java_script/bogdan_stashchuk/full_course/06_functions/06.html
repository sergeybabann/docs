6 Функции

Что такое функция?

Функция - это блок кода, который можно выполнять многократно. Давайте посмотрим на пример:

Папка 01. Пример неоптимизированного кода с повторяющимися блоками кода: 

                    
                        let a = 5
                        let b = 3
                        
                        let c 
                        
                        c = a + b
                        console.log(c) // 8 - одинаковые блоки кода
                        
                        a = 8
                        b = 12
                        
                        c = a + b
                        console.log(c) // 20 - одинаковые блоки кода

Примечание! Очень рекомендуется избегать повторения блоков кода.

В нашем примере кода есть 2 повторения блоков кода, в комментариях написано об этом. Если присутствуют такие повторения, то это значит, что код написан не оптимально, и его нужно оптимизировать. И это можно оптимизировать путем создания функции. И в этом примере можно создать функцию и назвать ее, например, sum Внутри этой функции мы будем суммировать значения двух переменных и печатать в консоль результат.

Папка 02. Пример оптимизированного кода с использованием функции: 

                    
                        let a = 5
                        let b = 3
                        
                        // функция 
                        function sum(a, b) {
                            const c  = a + b
                            console.log(c)
                        }
                        
                        sum(a, b) // 8
                        
                        a = 10
                        b = 12 
                        
                        sum(a, b) // 20

Мы перенесли повторяющийся блок внутрь функции.

Повторяющийся блок: 

                    
                        c = a + b
                        console.log(c) // 8 - одинаковые блоки кода

Оптимизированный код повторяющегося блока кода в функции: 

                    
                        const c  = a + b
                            console.log(c)

И при этом внутри функции мы объявляем переменную c при помощи const Потому, что она нужна только внутри функции. И в результате складывания 2-ух значений мы получаем новое значение и печатаем его в консоль.

И теперь мы можем вызвать функцию sum 

                    
                        sum(a, b) // 8 (вызов функции)
                    
                
                    
                        let a = 5
                        let b = 3
                        
                        // функция 
                        function sum(a, b) {
                            const c  = a + b
                            console.log(c)
                        }
                        
                        sum(a, b) // 8
                        
                        a = 10 
                        b = 12 
                        
                        sum(a, b) // 20

При повторном вызове функции ее значение будет отличаться от значения первого вызова. Потому, что переменным a и b были присвоены новые значения, которые при операции сложения дают и новый результат значения константы c

Тем самым функция позволяет выполнять одни и те же действия многократно с разными вводными значениями. Вот для этого как раз в функции и присутствуют параметры.

параметр a и b 

                    
                        function sum(a, b)

Какие могут быть функции?

Функция может быть:

  • именованной
  • присвоенна переменной
  • анонимной (не иметь имени)
  • аргументом при вызове другой функции
  • значением свойства объекта - и такое свойство называется метод

Функции в JavaScript - это объект. И соответственно как у любого объекта у функции есть свойства.

Папка 03. Пример функции 

                    
                        function myFn(a, b) {
                            let c
                            a = a + 1
                            c = a + b
                            return c
                        }

Функция myFn , у нее есть параметры a и b , есть тело функции и в ее теле выполняются определенные действия. Мы здесь объявляем переменную c , потом увеличиваем значение параметра a на единицу, и потом сумму параметров a и b мы присваиваем переменной c , и в результате еще и возвращаем значение переменной c

Если вы введете в консоли команду console.dir и передадите название этой функции как аргумент 

                    
                        console.dir(myFn)
                    
                
                    
                        function myFn(a, b) {
                            let c
                            a = a + 1
                            c = a + b
                            return c
                        }
                        console.dir(myFn)

, то вы увидите структуру этого объекта. Потому, что любая функция - это объект. И у такого объекта как myFn есть свойства:

  • arguments
  • caller
  • length - показывает количество параметров, в данном случае их 2 (a, b)
  • name (название функции - myFn)
Объявление и вызов функции

Папка 01. Структура функции 

                    
                        function myFn(a, b) {
                            let c
                            a = a + 1
                            c = a + b
                            return c
                        }

  • Ключевое слово function
  • Имя функции - myFn
  • Тело функции - между фигурными скобками
  • Параметры функции (a, b). Парметры функции ведут себя на самом деле как переменные. И в теле функции мы имеем доступ к этим переменным. На самом деле значения этих переменных определяются в момент вызова функции. И когда мы вызываем функцию, мы говорим, какие значения мы хотим присвоить параметрам - переменным внутри функции. Но эти переменные действуют только внутри функции. Если вы попытаетесь получить доступ к переменным a и b вне функции, то вы получите ошибку - переменная не определена.

В этом примере функция возвращает результат: 

                    
                        return c

Для возвращения результата используется ключевое слово return И мы здесь возвращаем результат, который равен значению переменной c

Здесь также еще один очень важный момент!

После инструкции с ключевым словом return , функция прекращает выполнения дальнейших инструкций. То есть, если после этой строки return есть какие-то операции, они не будут выполнены. Функция возвращает результат и заканчивает свою работу.

Если функция не имеет инструкции return , то она возвращает значение undefined (не определено), то есть функция всегда возвращает значение.

Если мы наберем и запустим в консоли браузера команду console.log() , то мы увидим результат выполнения функции log, а это функция, и этот результат undefined
log всегда возвращает undefined

Папка 02. Вызов функции 

                    
                        function myFn(a, b) {
                            let c
                            a = a + 1
                            c = a + b
                            return c
                        }

                        myFn(10, 3) // 14

Значения переменных определяются в момент вызова функции. Вызов функции:

                    
                        myFn(10, 3)

Так вот 10 и 3 в данном случае - это аргументы. Аргументы - это те значения, которые вы передаете в вызове функции. Параметры - это то, что находится в скобках, это названия параметров, которые находятся в скобках при объявлении функции. 

                    
                        function myFn(a, b) { // параметры при объявлении функции
                            let c
                            a = a + 1
                            c = a + b
                            return c
                        }

                        myFn(10, 3) // аргументы при вызове функции

То есть, если я говорю - аргументы функции, то я подразумеваю вызов функции. Если я говорю - параметры функции, то я говорю об объявлении функции. Аргументы меняются каждый раз при вызове одной и той же функции, если вводные параметры меняются, если вы меняете их с помощью аргументов. Названия параметров не меняются, они как были(a, b), так и остаются. И в данном примере, если мы вызываем функцию myFn с аргументами 10 и 3, то в результате параметр a получит значение 10, параметр b получит значение 3, в начале функции мы объявим новую переменную c , далее значение переменной a увеличивается на 1(как результат значение будет 11), дальше 11 будет добавлено к 3, и результат будет присвоен переменной c , это будет 14
И эта функция вернет это значение 14, это значение переменной c

Вы можете присвоить результат вызова этой функции переменной, и потом напечатать в консоль значение этой переменной, и вы получите 14

Рассмотрим шаг за шагом, что же происходит в данном примере: 

                    
                        function myFn(a, b) { // параметры при объявлении функции
                            let c
                            a = a + 1
                            c = a + b
                            return c
                        }

                        myFn(10, 3) // аргументы при вызове функции

При вызове функции происходит следующее: 

                    
                        myFn(10, 3) // вызов функции
  • Параметрам a и b присваиваются значения 10 и 3
  • Объявляется переменная с
  • Значение a увеличивается на 1
  • Сумма значений a и b присваивается переменной c
  • В конце возвращается значение переменной c с помощью ключевого слова return

Папка 03. Самая короткая функция: 

                    
                        function myFn() {}

                        myFn() // undefined

В этой самой короткой функции есть ключевое слово function , есть короткое имя myFn , далее нет параметров, но есть круглые скобки и они являются частью синтаксиса функции, и потом после круглых скобок идет пустое тело функкции - 2 фигурные скобки.

И конечно же если вы вызовите такую функцию:

                    
                        myFn() // undefined

, то она вернет undefined , потому что у нее нет аргументов. Вызов функции осуществляется без аргументов, по-скольку нет и параметров, поэтому смысла передавать аргументы нет. И поэтому она вернет undefined

Передача значения по ссылке

Папка 01. Передача объекта в качестве аргумента.

Рассмотрим пример, в котором в качестве аргумента в вызове функции мы передаем объект. 

                    
                        const personOne = {
                            name: 'Bob',
                            age: 21
                        }
                        
                        function increasePersonAge(person) {
                            person.age += 1 
                            return person 
                        }
                        
                        increasePersonAge(personOne) 
                        console.log(personOne.age)

В этом вызове personOne - это объект 

                    
                        increasePersonAge(personOne)

Объявлена функция increasePersonAge 

                    
                        function increasePersonAge(person) {
                        }

У этой функции есть параметр person, и внутри этой функции мы меняем свойство этого параметра при помощи оператора +=
Это оператор, который увеличивает значение того или иного свойства, либо другой переменной на 1
Мы берем и изменяем значение свойства age на 1

                    
                        person.age += 1

В итоге мы возвращаем person 

                    
                        return person

Мы подразумеваем, что person - это объект, который будет передаваться в функцию при вызове такой функции. И вот мы объявили объект personOne
В объекте есть 2 свойства name, age 

                    
                        const personOne = {
                            name: 'Bob',
                            age: 21
                        }

Смотрите что происходит:
Мы передаем в вызове функции аргумент personOne 

                    
                        increasePersonAge(personOne)

Это переменная, которую мы объявили ранее, и значение этой переменной - объект. Мы помним, что переменная содержит только ссылку на объект. То есть в вызове переменной мы передаем только ссылку на объект personOne
Тем самым personOne и person будут ссылаться на один объект в памяти. 

                    
                        increasePersonAge(personOne) 
                    
                
                    
                        function increasePersonAge(person)
                    
                
                    
                        const personOne = {
                            name: 'Bob',
                            age: 21
                        }
                        
                        function increasePersonAge(person) {
                            person.age += 1 
                            return person 
                        }
                        
                        increasePersonAge(personOne) 
                        console.log(personOne.age)

И этот объект имеет 2 свойста name, age

Что получается? Получается, что мы мутируем внешний объект, когда меняем одно из его свойств. 

                    
                        person.age += 1 // меняем свойство age

Так как person и personOne ссылаются на один и тот же объект в памяти, то соответственно personOne также изменится.

Получаем доступ к значению age 

                    
                        console.log(personOne.age) // 22

и видим его новое значение, было 21 стало 22

Это происходит из-за того, что мы передали объект по ссылке. Копии объекта не было создано. Внутри функции мы изменили внешний объект.

!Внимание. Внутри функции не рекомендуется мутировать внешние объекты. Это очень важная рекомендация. То есть, внутри функции не меняйте внешние объекты, которые передаются как аргументы.

Что делать, если вы хотите выполнять какие-то действия внутри функции с объектами?

Папка 02. Создание копии объекта. 

                    
                        const personOne = {
                            name: 'Bob',
                            age: 21
                        }
                        
                        function increasePersonAge(person) {
                            const updatedPerson = Object.assign({}, person)
                            updatedPerson.age += 1
                            return updatedPerson
                        }
                        
                        const updatedPersonOne = increasePersonAge(personOne)
                        console.log(personOne.age) // 21
                        console.log(updatedPersonOne.age) // 22

Вы можете внутри функции создавать копии объекта. То есть все также как и раньше. У нас есть объект personOne с двумя свойствами, также мы объявили функцию increasePersonAge(person) и у нее есть 1 параметр person 

                    
                        function increasePersonAge(person)

здесь мы вызываем функцию:

                    
                        const updatedPersonOne = increasePersonAge(personOne)

Обратите внимание на то, что результат вызова функции мы присваиваем новой переменной updatedPersonOne
Почему мы делаем это? Потому что внутри функции increasePersonAge мы создаем новый объект, который называется updatedPerson : 

                    
                        const updatedPerson = Object.assign({}, person)

И мы по сути создаем копию объекта на основании того объекта personOne, который передан в качестве аргумента (находится в круглых скобках в вызове): 

                    
                        const updatedPersonOne = increasePersonAge(personOne)

То есть person на входе в функцию будет ссылаться на объект personOne : 

                    
                        const updatedPerson = Object.assign({}, person)
                    
                
                    
                        function increasePersonAge(person) {
                            const updatedPerson = Object.assign({}, person)
                            updatedPerson.age += 1
                            return updatedPerson
                        }

Но с помощью Object.assign({}, person) мы создаем новый объект, и формируем его исходя из свойств объекта person : 

                    
                        Object.assign({}, person)

То есть updatedPerson теперь будет новым объектом. Это будет новая ссылка на новый объект: 

                    
                        const updatedPerson = Object.assign({}, person)

И дальше мы уже меняем этот новый объект. Мы меняем значение свойства age : 

                    
                        updatedPerson.age += 1

И в конце функии мы возвращаем новый объект updatedPerson как результат работы функции. 

                    
                        return updatedPerson

И результат такой функции мы присваиваем новой переменной: 

                    
                        const updatedPersonOne = increasePersonAge(personOne)

И тем самым мы получаем новый объект, который мы получили в результате вызова функции. И теперь в этом новом объекте свойство age будет равняться 22 : 

                    
                        console.log(updatedPersonOne.age) // 22

Но personOne.age не изменился: 

                    
                        console.log(personOne.age) // 21

Потому что мы не мутируем оригинальный объект.

Вот один вариант из того, как же поступать, если в функцию вы хотите передавать объекты и изменять их внутри функции, при этом не меняя оригинальный объект. Чтобы не было мутации.

Тем самым в итоге объект personOne не изменится. А мы будем менять только копию такого объекта.

Также нужно обратить внимание на то, что иногда функции создают специально именно таким образом, чтобы они меняли внешние объекты, хотя это делать не рекомендуется. Но иногда такое делают. В большинстве случаев рекомендуется создавать копии объектов и работать с ними внутри функции. То есть функции не должны менять внешние переменные, они должны работать только со своими внутренними переменными.

Колбэк функции

Папка 01. Посмотрим на пример. Здесть у нас есть 2 функции: 

                    
                        function anotherFunction() {
                            //...действия
                        }
                        
                        function fnWithCallBack(callbackFunction) {
                            callbackFunction()
                        }
                        
                        fnWithCallBack(anotherFunction)

В одной функции нет параметров, а в другой функции есть 1 параметр. И мы подразумеваем, что значением этого параметра, при вызове этой функции, будет другая функция. В теле функции с параметром мы вызываем эту функцию обычным образом используя круглые скобки. Вот так: 

                    
                        function fnWithCallBack(callbackFunction) {
                            callbackFunction()
                        }

callback функия - это функция, которая вызывает внутри себя другую функцию.

Когда мы создали функцию, в теле которой вызывается другая функция, мы можем вызвать эту фунцкию и передать ей другую функцию. И аргументом этой функции является другая функция. Вот так: 

                    
                        fnWithCallBack(anotherFunction)

Смысл callback функции в том, что она вызывается внутри другой функции. Вот и все!

И конечно без вызова функции fnWithCallBack, функция anotherFunction не вызывется. Потому, что вызывается она только внутри функции fnWithCallBack.

В данном примере функция anotherFunction - это callback функция. Она передается как агрумент в другую функкцию.

Папка 02. Рассмотрим другой пример 

                    
                        function printMyName() {
                            console.log('Bogdan')
                        }
                        
                        setTimeout(printMyName, 1000)

Есть функция, которая называется printMyName. У нее нет параметров, но есть вывод в консоль. И далее мы вызываем функцию setTimeout. Эта функция встроена в JavaScript, то есть она доступна по умолчанию, как один из методов глобального объекта window в браузере либо global в Nodejs.

Так вот, эта встроенная функция внутри себя вызывает функцию, которая передана как 1-ый аргумент. И мы нигде не вызываем эту функцию printMyName, потому что она вызывется внутри функции setTimeout.

В функцию setTimeout передаем 2 аргумента. 1-ый - это название функции, 2-ой - это число в миллисекундах. И setTimeout работает таким образом, что она вызывает callback функцию printMyName, через определенное время. В данном примере через 1000 миллисекунд, либо 1 секунду. Вот так работает callback функция.

И у нас здесь происходит то, что текст вызова concole.log мы увидим в консоли через 1 секунду. И запустим этот код в терминале.

А теперь попробуем сделать следующее. Добавим еще один вызов в консоль для более явного отслеживания. 

                    
                        function printMyName() {
                            console.log('Bogdan')
                        }
                        
                        console.log('Start')

                        setTimeout(printMyName, 1000)

Можно поиграться с временем интервала. Если навести курсор на название встроенной функции, то вы увидите какие аргументы ожидает эта функция.

Правила работы с функциями

  • Называйте функции исходя из задач. Чтобы было понятно что функция делает.
  • Одна функция должна выполнять одну задачу.
  • Не рекомендуется изменять внешние относительно функции переменные. То есть, если функция работает с объектами, то внутри функции создавайте копии этих объектов и меняйте их, не меняя при этом оригинальные объекты, которые были переданы как значение аргументов. Иными словами функция должна быть ЧИСТОЙ. То есть чистая фунция работает только с переменными, которые были объявлены внутри самой функции. И она не меняет внешние переменные, даже те, которые переданы как аргументы этой функции. Такая функция называется - Чистая функция.
