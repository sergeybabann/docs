Введение в классы и прототипы

Что позволяют делать классы?

Классы позволяют создавать прототипы для объектов.

Допустим есть производство автомобилей, и на этом производстве можно создавать различные модели, разных цветов, разной длины и ширины, с разными двигателями. И на этом производстве можно создать любой экземпляр автомобиля. И этому производству можно дать, например, такие параметры такие, как цвет машины, мощность двигателя, и производство создаст экземпляр автомобиля и отдаст его покупателю.

Конкретный экземпляр будет иметь собственные свойства такие, как цвет и остальные характеристики. И кроме этого, такой экземпляр будет наследовать все методы автомобиля такие, как ехать, поворачивать, останавливаться и так далее.

На основании прототипов создаются экземпляры. С одного прототипа (заготовки) можно создать много экземпляров объекта. И все экземпляры будут независимы друг от друга, и каждый экземпляр может иметь свои собственные свойства и методы. Каждый экземпляримеет свои собственные свойства и методы, которые привязаны только к конкретному экземпляру.

Экземпляры наследуют методы и свойства прототипов. То есть могут быть собственные методы, а могут быть унаследованные методы.

Примеры класса

В JavaScript понятие классов это всего лишь синтаксическая надстройка над функциями и объектами.

На самом деле экземпляры классов в JavaScript создаются с помощью вызова функции. Класс - это шаблон для создания экземпляров класса.

В примере ниже есть класс, который называется Comment. Между фигурными скобками находится все, что касается конкретного класса. А там находятся свойства и методы. В данном примере есть 2 метода такие, как constructor и upvote. Эти методы похожи на методы в объектах, синтаксис такой же. Идет название метода, далее в скобках находятся опциональные параметры, и потом тело конкретного метода.

Что здесь важно отметить? В этом классе есть переменная this, это специальная переменная, которая указывает на экземпляр класса. Такой класс - это всего лишь шаблон для создания новых объектов. И когда вы будете создавать новый экземпляр такого класса, то this автоматически будет ссылаться на новый экземпляр. И ссылаясь на конкретный экземпляр класса, вы можете соответственно внутри класса (внутри методов и свойств класса) менять собственные свойства и методы конкретного экземпляра. 

                    
                        class Comment {
                            constructor(text) {
                        this.text = text
                        this.votesQty = 0
                            }
                        upvote() {
                            this.votesQty += 1
                            }
                        }

В методе constructor есть 2 присваивания. 

                    
                        this.text = text
                        this.votesQty = 0

В методе constructor есть 1 параметр (text)

При вызове такого метода, который называется constructor мы передаем значение для параметра (text) 

                    
                        constructor(text)

Далее внутри этого конструктора мы конкретному экземпляру этого класса Comment добавляем свойство text 

                    
                        this.text

И значение этого свойства text будет значение параметра text, которое было передано в качестве аргумента в вызове этого метода. 

                    
                        this.text = text

То есть у нового экземпляра этого класса появится собственное свойство text с каким то текстовым значением, которое мы передали в вызове этой функции constructor

Таким же образом мы добавили свойство votesQty, и справа от знака присваивания стоит цифра 0, почему 0? Конструктор вызывается только тогда, когда вы создаете новый экземпляр конкретного объекта. 

                    
                        this.votesQty = 0

Мы хотим создать новый комментарий с каким то текстом, и у этого комментария будет определенное количество голосов. То есть можно будет голосовать за этот комментарий, и можно будет повышать количество голосов. Когда мы создаем новый экземпляр комментария, у него не будет еще голосов. Соответственно начальное количесвто голосов 0. Потому и в конструкторе мы это указываем путем создания нового свойства votesQty и добавлением начального значения этому свойству. То есть начальное количество голосов у каждого комментария будет 0.

Далее идет метод upvote
Это функция, и внутри этой функции мы увеличиваем значение свойства votesQty, и также обращаемся к конкретному экземпляру комментария (класса комментария) - this, берем его свойство votesQty и увеличиваем это значение на 1, используя оператор +=

                    
                        upvote() {
                            this.votesQty += 1
                            }

Можно также написать другой вариант присваивания: 

                    
                        this.votesQty = this.votesQty + 1

Теперь после создания экземпляра класса можно будет увеличивать количесвто голосов для конкретного комментария, который является экземпляром класса комментарий, путем вызова метода upvote, и этот метод находится на уровне класса. Он не будет присвоен как отдельный метод конкретного экземпляра этого комментария, этого класса комментарий. Но он будет наследоваться всеми экземплярами этого класса комментарий.
