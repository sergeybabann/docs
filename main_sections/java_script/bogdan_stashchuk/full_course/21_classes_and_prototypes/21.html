21 Классы и прототипы

Введение в классы и прототипы

Что позволяют делать классы?

Классы позволяют создавать прототипы для объектов.

Допустим есть производство автомобилей, и на этом производстве можно создавать различные модели, разных цветов, разной длины и ширины, с разными двигателями. И на этом производстве можно создать любой экземпляр автомобиля. И этому производству можно дать, например, такие параметры такие, как цвет машины, мощность двигателя, и производство создаст экземпляр автомобиля и отдаст его покупателю.

Конкретный экземпляр будет иметь собственные свойства такие, как цвет и остальные характеристики. И кроме этого, такой экземпляр будет наследовать все методы автомобиля такие, как ехать, поворачивать, останавливаться и так далее.

На основании прототипов создаются экземпляры. С одного прототипа (заготовки) можно создать много экземпляров объекта. И все экземпляры будут независимы друг от друга, и каждый экземпляр может иметь свои собственные свойства и методы. Каждый экземпляр имеет свои собственные свойства и методы, которые привязаны только к конкретному экземпляру.

Экземпляры наследуют методы и свойства прототипов. То есть могут быть собственные методы, а могут быть унаследованные методы.

Примеры класса

В JavaScript понятие классов это всего лишь синтаксическая надстройка над функциями и объектами.

На самом деле экземпляры классов в JavaScript создаются с помощью вызова функции. Класс - это шаблон для создания экземпляров класса.

В примере ниже есть класс, который называется Comment. Между фигурными скобками находится все, что касается конкретного класса. А там находятся свойства и методы. В данном примере есть 2 метода такие, как constructor и upvote. Эти методы похожи на методы в объектах, синтаксис такой же. Идет название метода, далее в скобках находятся опциональные параметры, и потом тело конкретного метода.

Что здесь важно отметить? В этом классе есть переменная this, это специальная переменная, которая указывает на экземпляр класса. Такой класс - это всего лишь шаблон для создания новых объектов. И когда вы будете создавать новый экземпляр такого класса, то this автоматически будет ссылаться на новый экземпляр. И ссылаясь на конкретный экземпляр класса, вы можете соответственно внутри класса (внутри методов и свойств класса) менять собственные свойства и методы конкретного экземпляра. 

                    
                        class Comment {
                            constructor(text) {
                        this.text = text
                        this.votesQty = 0
                            }
                        upvote() {
                            this.votesQty += 1
                            }
                        }

В методе constructor есть 2 присваивания. 

                    
                        this.text = text
                        this.votesQty = 0

В методе constructor есть 1 параметр (text)

При вызове такого метода, который называется constructor мы передаем значение для параметра (text) 

                    
                        constructor(text)

Далее внутри этого конструктора мы конкретному экземпляру этого класса Comment добавляем свойство text 

                    
                        this.text

И значение этого свойства text будет значение параметра text, которое было передано в качестве аргумента в вызове этого метода. 

                    
                        this.text = text

То есть у нового экземпляра этого класса появится собственное свойство text с каким то текстовым значением, которое мы передали в вызове этой функции constructor

Таким же образом мы добавили свойство votesQty, и справа от знака присваивания стоит цифра 0, почему 0? Конструктор вызывается только тогда, когда вы создаете новый экземпляр конкретного объекта. 

                    
                        this.votesQty = 0

Мы хотим создать новый комментарий с каким то текстом, и у этого комментария будет определенное количество голосов. То есть можно будет голосовать за этот комментарий, и можно будет повышать количество голосов. Когда мы создаем новый экземпляр комментария, у него не будет еще голосов. Соответственно начальное количесвто голосов 0. Потому и в конструкторе мы это указываем путем создания нового свойства votesQty и добавлением начального значения этому свойству. То есть начальное количество голосов у каждого комментария будет 0.

Далее идет метод upvote
Это функция, и внутри этой функции мы увеличиваем значение свойства votesQty, и также обращаемся к конкретному экземпляру комментария (класса комментария) - this, берем его свойство votesQty и увеличиваем это значение на 1, используя оператор +=

                    
                        upvote() {
                            this.votesQty += 1
                            }

Можно также написать другой вариант присваивания: 

                    
                        this.votesQty = this.votesQty + 1

Теперь после создания экземпляра класса можно будет увеличивать количесвто голосов для конкретного комментария, который является экземпляром класса комментарий, путем вызова метода upvote, и этот метод находится на уровне класса. Он не будет присвоен как отдельный метод конкретного экземпляра этого комментария, этого класса комментарий. Но он будет наследоваться всеми экземплярами этого класса комментарий.

Создание экземпляров класса

Как создать новый экземпляр такого класса комментарий?

Создать новый экземпляр класса очень просто, используя ключевое слово new

new - это префиксный унарный оператор.

Мы создаем новую перменную, которая назывется firstComment
После знака присваивания стоит синтаксис new Comment, здесь Comment написан также как и в именовании класса Comment, с большой буквы. Далее в скобках идет вызов функции, в скобках стоит значение для параметра text, который находится в классе в скобках после метода constructor.

Папка 01

                    
                        class Comment { // класс Comment
                            constructor(text) {
                                this.text = text
                                this.votesQty = 0
                            }

                            upvote() {
                                this.votesQty += 1
                            }
                        }
                    
                
                    
    const firstComment = new Comment('First comment') // новая переменная

С помощью такого синтаксиса - new Comment('First comment') вызывается конструктор с 2-мя this, конструктор это функция, которая создает новый экземпляр объекта.

Для того, чтобы создать новый экземпляр объекта мы должны передать текстовое значение ('First comment') для свойства text, которое стоит сразу после this в конструкторе - this.text, для конкретного экземпляра класса комментарий.

И в итоге такая функция конструктор вернет новый объект. И этот новый объект будет наследовать все методы класса комментарий. И комментарий будет прототипом для такого экземпляра.

Папка 02. Свои собственные свойства

После создания нового экземпляра класса комментарий у переменной firstComment появятся свои собственные свойства. 

                    
                        class Comment { // класс Comment
                            constructor(text) {
                                this.text = text
                                this.votesQty = 0
                            }
                        
                            upvote() {
                                this.votesQty += 1
                            }
                        }
                        
                        const firstComment = new Comment('First comment') // новая переменная
                        
                        console.log(firstComment)

Выведем в консоль console.log(firstComment) и посмотрим на структуру. Развернем Comment и посмотрим. У этого объекта есть собственные свойства - это текст и количество голосов.

  • text: 'First comment'
  • votesQty: 0

Это те свойства, которые мы добавили к конкретному экземпляру объекта в конструкторе. Это собственные свойства этого конкретного объекта firstComment, который является экземпляром класса Comment 

                    
    const firstComment = new Comment('First comment') // экземпляр класса Comment

Здесь в этом примере вызова console.log(firstComment) в консоли браузера значение свойства text будет First comment, text: 'First comment' - это то, что мы передали в вызове конструктора: 

                    
    const firstComment = new Comment('First comment') // вызов конструктора

и начальное количество голосов 0, votesQty: 0
То есть пока еще функция upvote не вызывалась. И обратите внимание, что метода upvote у объекта firstComment нет. Но если раскрыть значение свойства prototype, которое находится между 2-мя квадратными скобками (в выводе в консоли веб-браузера), здесь вы найдете метод upvote. То есть этот метод находится на уровне класса Comment. И этот метод наследуется всеми экземплярами этого класса Comment

Соответственно можно вызывать такой метод как метод переменной firstComment. То есть получается что эти 2 свойства

  • text: "First comment"
  • votesQty: 0

- собственные свойства объекта firstComment, а вот метод upvote является унаследованным методом с родительского класса Comment. То есть нет необходимости добавлять метод upvote каждому экземпляру класса Comment. Достаточно просто унаследовать такой метод всеми экземплярами конкретного класса. Поэтому в выводе консоли браузера метод upvote виден на уровне прототипа.

Папка 03. Что такое наследование по цепочке?

Если после создания переменной firstComment, в консоли написать firstComment и поставить после нее точку - firstComment. , то вы увидите все возможные методы и свойства, которые доступны для использования на уровне объекта firstComment.

Так вот здесь вы увидите собственные свойства такие, как text и votesQty. Далее на уровне родительского класса Comment вы увидите, что доступны такие методы, как constructor и upvote. То есть можно вызвать upvote на уровне этого объекта firstComment, потому что upvote наследуется всеми экземплярами класса Comment.

Кроме этого вы видите в списке методы объекта. Почему они здесь видны? Потому, что комментарий наследует все методы и свойства объекта. А объект - это класс, который присутствует в JavaScript по умолчанию, и любые объекты JavaScript наследуют все методы этого родительского корневого класса Объект.
