9 Логические операторы

Оператор НЕ

Оператор !(не) или оператор !(отрицания)

Оператор ! чаще всего используется в условных инструкциях.

Посмотрим примеры с оператором НЕ: 

                    
                        !10 // false 
                        !0 // true 
                        !'abc' // false
                        !'' // true
                        !true // false
                        !undefined // true
  • !10 // false (не 10)
  • !0 // true (не 0, ноль это ложное значение)
  • !'abc' // false (не пустая строка, пустая строка это ложное значение)
  • !'' // true (не пустая строка, пустая строка это ложное значение)
  • !true // true (не true)
  • !undefined // true (не undefined)

Оператор !(не) всегда дает 1 из 2-ух значений. Либо true, либо false.

Оператор (не) - это префиксный оператор, и это унарный оператор.

Отрицание отрицания !!

Посмотрим на примеры с 2-мя знаками восклицания. 

                    
                        !!10 // 
                        !!0 //  
                        !!'abc' //
                        !!'' // 
                        !!true // 
                        !!undefined //

Используя двойной оператор !!(НЕ), можно легко конвертировать любое значение в булевое значение. Тем самым вы можете легко проверять ложность значения.

Ложные значения:

  • 0 - ноль
  • '' - пустая строка(2 одинарные кавычки)
  • undefined - не определено

Потому двойное отрицание всегда даст вам false на выходе:

  • !!0 // false
  • !!'' // false
  • !!undefined // false

Если исходное значение правдиво либо истинно, то двойное отрицание даст нам всегда true

Оператор короткого замыкания И

Операторы && и || являются операторами короткого замыкания.

Оператор &&

Выражение 1 && Выражение 2. В этом выражении есть оператор &&(и), он находится между 2-мя операндами. Каждый из операндов - это выражение, а любое выражение возвращает значение. Если выражение 1 ложно, то выражение 2 игнорируется. И возвращается результат выражения 1, как результат всего этого выражения:

                    
                        Выражение полностью: Выражение 1 && Выражение 2

То есть если результат выражения 1: undefined, то результат всего этого выражения будет undefined.

Но ключевой момент здесь в том, что выражение 2, в таком случае, вообще не оценивается интерпретатором JavaScript.

Оператор && требует правдивости обоих операндов, для того,чтобы результирующее выражение было правдиво. Потому нет смысла оценивать результат выражения 2, если стало понятно что результат будет ложным. Потому возвращается сразу результат выражения 1, что и происходит. Потому оператор && называется оператором короткого замыкания.

Но если выражение 1 истинно, то оценивается выражение 2. Если выражение 2 ложно, то возвращается результат этого выражения. Если выражение 2 истинно, то также возвращается результат выражения 2.

Оператор короткого замыкания ИЛИ

Оператор || (или)

У нас есть выражение, состоящее из 2-ух выражений. И между 2-мя операндами находится инфиксный оператор || (или) 

                    
                        выражение 1 || выражение 2

Наша задача оценить это выражение и получить результат этого выражения.

Если выражение 1 истинно, то выражение 2 игнорируется, и возвращается результат выражения 1, и это будет результат всего выражения полностью. Потому, что оператор || (или) - это оператор короткого замыкания. Как только требования оператора || удовлетворены, то естькогда мы нашли 1-ое истинное выражение, мы возвращаем результат такого выражения.

Пример:

Откроем консоль веб-браузера и напишем такой код: 

                    
                        'Bogdan' || 'Default value'

Результатом такого выражения будет строка: 'Bogdan', потому что 1-ый операнд (1-ое выражение) истинно. И потому возвращается результат 1-ого выражения.

Другой пример:

                    
                        '' || 'Default value'

1-ое выражение - это пустая строка. Результатом будет строка 'Default value', потому что 1-ый операнд ложный. И поэтому оценивается 2-ой операнд (2-ое выражение). И это будет результат всего этого выражения.

Именно такая конструкция используется тогда, когда вы хотите присвоить какой-то переменной дефолтное значение, если значение у другой переменной, на основании которой вы хотите присвоить значение, отсутствует.

Пример

напишем в консоли такой код: 

                    
                        a || 'Default value'

И мы получим ошибку: VM112:1 Uncaught ReferenceError: a is not defined at anonymous:1:1

Потому, что мы попытались получить доступ к переменной, которая не была объявлена. Следовательно дальше мы не пошли и не посмотрели на 2-ое выражение после оператора ||

Трюк с оператором И и вызовом функции

Пример

Наберем в консоли веб-браузера такой код: 

                    
                        let b = 10

Значение переменной b - это 10, и оно истинно. Вернется undefined

В этой же сесси продолжаем писать код, и напишем следующее: 

                    
                        b && console.log('Выполнено!')

Возвращается undefined, и это результат всего этого выражения.

Получается следующее:
Сначала был найден результат выражения, которое находится слева (левый операнд b), а результат этого выражения 10 (мы его присвоили в это сессии). Потому мы пошли дальше (здесь мы используем оператор &&(и)). И мы выполнили 2-ую часть этого выражения(правый операнд), а это выражение возвращает undefined (это то, что возвращает метод log). И кроме того, мы увидели строку Выполенено!(потому, что это то, что выполняется внутри метода log).

Продолжаем писать код в этой же сессии. И напишем такой код: 

                    
                        let c

Объявили переменную с и нажимаем enter, значение не присваиваем получаем undefined. Пишем далее: 

                    
                        с && console.log('Выполнено!')

Я не вижу вывода в консоль строки 'Выполнено!', почему? Потому, что оператор &&(и) - это оператор короткого замыкания. Как только мы нашли ложное значение, мы его вернули, и дальше на остальные операнды мы не смотрим. То есть функция не была вызвана. Тем самым, используя такой подход вы можете условно вызывать те или иные функции. Потому, что здесь на месте второго выражения(2-ого операнда) может быть любое выражение, а не только console.log

И вы хотите вызывать это выражение зависимо от того, какие значения есть у тех или иных других переменных. Зависимо от других условий. Конечно то же самое можно написать с помощью инструкции if, но тогда у вас будет несколько строк кода. А в этом случае у вас одна строка. То есть это такой трюк, который часто используется в JavaScript.

У автора курса в этом примере результат будет undefined, у меня же появляется ошибка: Uncaught ReferenceError: с is not defined at anonymous:1:1
