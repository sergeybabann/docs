4 Объекты

Объекты

У объектов есть свойства. Если какое-то свойство содержит функцию как значение, такое свойство называется метод. Практически все сущности в JavaScript - это объекты. Объект - это набор свойств, имя:значение. Объект - это тип значений, также тип переменных, то естьесли вы присваиваете значение типа объект переменной, то можно сказать что эта переменная является объектом. Либо ее тип это объект. Объект также это ссылочный тип. Переменная, которой вы присваиваете объект содержит только ссылку на объект, который находится в другом месте памяти компьютера.

Объект и его структура 

                    
                        // объект
                        const myCity = {
                            city: 'New York',
                            popular: true,
                            country: 'USA'
                        }

Объект записывается между фигурными скобками, в этом объекте 3 свойства:

  • popular
  • city
  • country

Справа после двоеточия вы видете значения соответствующих свойств. Пара ключ:значение разделяются запятой. Справа значение свойства, а слева название свойства. В объекте рекомендуется применять const , когда вы объявляете переменную, которая будет содержать объект как свое значение.

Порядок свойств в объекте не имеет значения: 

                    
                        // объект
                        const myCity = {
                            city: 'New York',
                            popular: true,
                            country: 'USA'
                        }
                    
                
                    
                        // объект
                        const myCity = {
                            popular: true,
                            city: 'New York',
                            country: 'USA'
                        }

Два объекта с одинаковыми свойствами, разница между ними лишь в том, что эти свойства идут в разных порядках. В JavaScript эти 2 объекта технически одинаковые.

Папка 01. Получение значений свойств объекта:

                    
                        // объект
                        const myCity = {
                            city: 'New York',
                            popular: true,
                            country: 'USA'
                        } 
                        
                        console.log(myCity.city) // 'New York'
                        
                        console.log(myCity.popular) // true

Получать доступ к значениям свойств объекта можно при помощи точки после названия обекта. После метода log объекта console идут скобки, в них есть название объекта и название свойства объекта. Между ними стоит точка. Название объекта - точка - название свойства объект. Этот синтаксис называется - точечная запись. (dot notation)

Папка 02. Изменение значений свойств объекта 

                    
                        const myCity = {
                            city: 'New York',
                        }
                        
                        myCity.city = 'Las Vegas'
                        console.log(myCity)
                        // {city: 'Las Vegas'}

Используя точечную запись вы можете менять значения свойств объекта. Значение свойства myCity сначала было 'New York' , а затем стало 'Las Vegas' ., при помощи оператора присваивания (=) мы изменили значение этого свойства.

Создание и удаление свойств объектов

Папка 01. Добавление новых свойств в объект 

                    
                        const myCity = {
                            city: 'New York'
                        }
                        
                        myCity.popular = true // добавление нового свойства в объект 
                        при помощи этого выражения
                        
                        console.log(myCity) 
                        // {city: 'New York', popular: true} 
                        
                        myCity.country = 'USA' // добавление нового свойства в объект 
                        при помощи этого выражения
                        
                        console.log(myCity)
                        // {city: 'New York', popular: true,  country: 'USA'}

Мы объявили переменную используя const , присваивать новое значение переменной, которая объявлена при помощи const нельзя. В этом примере мы изменяем объект, объект, который является значением переменной myCity . Сначала у объекта было одно свойство - myCity , а в итоге других операций у объекта myCity будет уже 3 свойства: city, popular, country

Очень важный нюанс, который мы опишем подробно

В данном примере мы не меняем значение переменной myCity , то есть мы не присваиваем ей новое значение. Мы меняем объект, а сама переменная, как содержала ссылку на объект, так и содержит. Эта ссылка не меняется. И это главное для const , мы не присваиваем новое значение, и значение переменной myCity не меняется. Все что мы делаем, мы переходим по ссылке в область памяти, где находится этот объект, и меняем этот объект при помощи операций добавления новых свойств в объект, используя точечную запись. Можно сказать мы мутируем этот объект, не меняя значение самой переменной myCity.

Тем самым мы можем легко объявлять переменные типа - объект, используя const , и смело менять содержимое таких объектов. Добавлять и удалять новые свойства, менять их значения и так далее. Все это возможно благодаря тому, что объект - это ссылочный тип в JavaScript.

Папка 02. Удаление свойств из объекта 

                    
                        // объект
                        const myCity = {
                            city: 'New York',
                            popular: true,
                            country: 'USA'
                        } 
                        
                        delete myCity.country // операция удаления свойства из объекта
                        
                        console.log(myCity) 
                        // {city: 'New York', popular: true}

delete - Оператор удаления
В данном примере в объекте myCity есть 3 свойства.

Таким образом при помощи оператора delete(оператора удаления) мы удалили из объекта myCity свойство country
Выводим в консоль объект myCity и увидим, что в объекте осталось всего 2 свойства. При помощи данной операции мы можем удалять любое свойство из объекта.

Папка 03. Доступ к значению свойства с использованием скобок. bracket notation (скобочная запись) 

                    
                        const myCity = {
                            city: 'New York'
                        }
                        
                        myCity['popular'] = true // добавление свойства в объект
                        
                        console.log(myCity) 
                        // {city: 'New York', popular: true}
                        
                        const countryPropertyName = 'country' 
                        
                        myCity[countryPropertyName] = 'USA'
                        
                        console.log(myCity) 
                        // {city: 'New York', popular: true, country: 'USA'}

Кроме точечной записи, вы можете работать со свойствами объекта, используя квадратные скобки. В данном примере тот же объект myCity 

                    
                        myCity['popular'] = true // добавление свойства в объект

В скобках указывается свойство, которое вы хотите добавить, и оно заключено в кавычки. Выводим свойство в консоль и видим в объекте 2 свойства.

Такой синтаксис с квадратными скобками используется в том случае, если название свойства является значением той или иной переменной. И это указано в примере ниже: 

                    
                        const countryPropertyName = 'country' 
                        
                        myCity [countryPropertyName] = 'USA'

Допустим вы создаете переменную, которая называется 

                        
                            countryPropertyName = country

Значение этой переменной - country
И допустим вы хотите добавить новое свойство к объекту myCity , которое будет называться так, как значение переменной countryPropertyName , и это значение в данном случае - country., и как раз в этом случае вам подойдет только синтаксис с квадратными скобками.

Вы не можете написать выражение 

                    
                        myCity.countryPropertyName

Потому, что в таком случае создастся свойство, которое называется countryPropertyName ., нам же необходимо создать свойство, которое называется country

И мы пишем нужное нам выражение в котором сначала идет назание объекта, затем в квадратных скобках идет название переменной: 

                    
                        myCity[countryPropertyName] = 'USA'

JavaScript сначала перед созданием нового свойства посмотрит какое значение у countryPropertyName , а значение у нас country (смотрим пример ниже) 

                    
                        const countryPropertyName = 'country' 
                        
                        myCity[countryPropertyName] = 'USA'

Именно с именем country создастся новое свойство, значение которого в данном случае будет 'USA'
Выводим этот объект в консоль, и увидим в объекте уже 3 свойства: 

                    
                        const myCity = {
                            city: 'New York'
                        }
                        
                        myCity['popular'] = true // добавление свойства в объект
                        
                        console.log(myCity) 
                        // {city: 'New York', popular: true}
                        
                        const countryPropertyName = 'country' 
                        
                        myCity[countryPropertyName] = 'USA'
                        
                        console.log(myCity) 
                        // {city: 'New York', popular: true, country: 'USA'}

Скобочная запись отличается от точечной записи тем, что в квадратных скобках вы можете по сути использовать любое выражение JavaScript ., в данном примере мы просто использовали название переменной, содержащей строковое значение country, и это же значение мы добавляем уже как название нового свойства со значением 'USA' ., и все это мы делаем при помощи скобочной записи bracket notation

Вложенные свойства

У объектов JavaScript могут быть вложенные свойства, которые содержат вложенные объекты. В данном примере у объекта myCity есть свойство info, и значение этого свойства это - объект, который имеет внутри свои 2 свойства:

  • isPopular
  • country

Папка 01. 

                    
                        const myCity = {
                            city: 'New York',
                            info: {
                                isPopular: true,
                                country: 'USA'
                            }
                        }
                        
                        console.log(myCity.info.isPopular)
                        // true

Такой объект называется вложенным объектом. И у этих свойств также могут быть объекты, как значения. Тем самым вложенность может быть многоуровневая.

Например, если вы хотите получить доступ к свойству isPopular вложенного объекта info, то вам необходимо использовать теперь 2 точки для того, чтобы дойти по иерархии объекта до нужного значения. Смотрим пример: 

                    
                        console.log(myCity.info.isPopular) 
                        // true

И в консоли мы увидим true ., значение свойства isPopular
Точно также можно удалять свойства, используя либо точечную запись, либо скобочную.

Папка 02. Удаление свойства из объекта

В примере ниже вы видете как при помощи оператора delete происходит удаление свойства isPopular , которое находится в объекте info ., К объекту info вы должны дойти через доступ с помощью точки соответственно, и в квадратных скобках прописать нужное свойство для его удаления, заключенное в кавычки. 

                    
                        delete myCity.info['isPopular']

И смотрим весь пример кода: 

                    
                        const myCity = {
                            city: 'New York',
                            info: {
                                isPopular: true,
                                country: 'USA'
                            }
                        }
                        
                        console.log(myCity.info.isPopular)
                        // true
                        
                        delete myCity.info['isPopular']
                        
                        console.log(myCity)
                        // {city: 'New York', info: {country: 'USA'}}

Примечание!
Используйте скобочную запись только тогда, когда в скобках должнобыть какое-то выражение. Например название переменной.

Сокращенный формат записи свойств

Папка 01. Использование переменных при формировании объекта 

                    
                        const name = 'Bogdan'
                        const postsQty = 23

В данном примере (выше) у нас есть две переменные: name и postsQty

Допустим теперь мы хотим создать новый объект с названием userProfile, и у этого объекта будут свои свойства. 

                    
                        const userProfile = {
                            name: name,
                            postsQty: postsQty,
                            hasSignedAgreement: false 
                        }

Что в итоге мы имеем и о чем нужно сказать?

Значения в созданном объекте для свойств name и postsQty сформированы исходя из значений переменных, объявленных ранее. Вы можете легко использовать переменные в качестве значений для свойств объекта. И в данном примере эти значения переменных, которые были объявлены ранее выступают в качестве значений свойств уже нового объекта. Вот полный пример кода: 

                    
                        const name = 'Bogdan'
                        const postsQty = 23
                        
                        const userProfile = {
                            name: name,
                            postsQty: postsQty,
                            hasSignedAgreement: false 
                        }

Более того, в качестве значений свойств объекта можно использовать любые выражения. И сначала JavaScript получит результат того или иного выражения, а после этого этот результат присвоится тому или иному свойству объекта. В данном случае мы просто используем название переменных как выражение.

Обратите внимание - в данном объекте userProfile свойство name имеет значение переменной name ., имена совпадают. Совпадает имя свойства с именем переменной. И тоже самое мы видим в переменной postsQty

Примечание!
Имя свойства совпадает с именем переменной, которую мы используем для получения значения для этого свойства.

В JavaScript есть возможность сокращенной записи при формировании таких объектов. Пример:

Папка 02. Сокращенный формат записи свойств объекта: 

                    
                        const name = 'Bogdan'
                        const postsQty = 23
                        
                        const userProfile = {
                            name,
                            postsQty,
                            hasSignedAgreement: false 
                        }

Сокращенный формат записи свойств можно применять в объекте, если вы формируете те или иные свойства исходя из переменных, которые были объявлены ранее. Такой формат создания объектов очень рапространен в JavaScript. Обычно часть значений уже известна до формирования объекта, и эти значения были присвоены определенным переменным. И при создании объекта вам нужно всего лишь сослаться на значения этих переменных и использовать их для получения значений свойств объекта.

Примечание!
Сокращенные свойства рекомендуется размещать в начале объекта. Пототму, что так удобнее читать объект. Чтобы было еще удобнее читать объект, свойства можно сортировать по их длине, короткие сверху и длинные ниже.
