Циклы

Папка 01. Для чего нужны цыклы?

В коде есть команды, в этих командах есть 2 выражения, а именно console.log(i) и i++

                    
                        let i = 0
                        console.log(i)
                        i++ 

                        console.log(i)
                        i++

                        console.log(i)
                        i++

                        console.log(i)
                        i++

                        console.log(i)
                        i++

Мы по сути дублируем блоки кода. У нас есть повторяющиеся действия. Блок кода повторяется многократно. Смысла писать так код нет, поэтому для этого используются циклы в JavaScript.

Папка 02. Второй пример. Массив. Перебор всех элементов массива без цикла. 

                    
                        // Перебор всех элементов массива без цикла
                        const myArray = [true, 'abc', 10]
                        
                        console.log(myArray[0])
                        console.log(myArray[1])
                        console.log(myArray[2])

Но так делать не удобно, если массив большой, например, на 1000 элементов, писать 1000 строк для того, чтобы перебрать все элементы массива это нецелесообразно. Поэтому существуют другие варианты перебора элементов массива, то есть делать это в циклах.

Папка 03. Перебор всех свойств объекта без цикла

Есть объект и в нем есть 3 свойства: x, y, z 

                    
                        // перебор всех свойств объекта без цикла
                        const myObject = {
                            x: 10,
                            y: true,
                            z: 'abc'
                        }
                        
                        console.log(myObject.x)
                        console.log(myObject.y)
                        console.log(myObject.z)

Здесь также идет перебор свойств объетка без цикла, используюя многократное повторение вызова метода log - console.log(), и в скобках аргументами передавать разные свойства.

Но так делать тоже не удобно. Перебирать свойства объекта в ручную. Для удобства перебора как свойств объекта, так и элементов массива используются циклы.

Цикл for

Все цыклы - это инструкции.

В цикле for есть зарезервированное слово for, с него начинается цикл. Далее в скобках находятся 3 исходные инструкции. 1) Начальная инструкция, 2) Условие 3) Итерационное действие. Далее находится блок кода, который выполняется на каждой итерации.

                    
                        for (let i = 0; i < 5; i++) {
                            console.log(i)
                        }

2-ая и 3-тья инструкции - это по сути выражения-инструкции.

Рассмотрим подробнее работу цикла.

console.log(i) - этот блок кода выпоняется много раз. Значения (i) в выводе в консоли браузера меняются, идет от 0 до 4.

  • Начинается цикл тем, что мы объявляем переменную i и даем ей начальное значение 0
    let i = 0;
  • Далее 0 сравнивается с цифрой 5
    0 < 5 (ноль меньше 5), то есть это условие правдиво и потому идет выполнение следущего блока кода, выводится в консоль знчение переменной i и видим в консоли брузера 0
    console.log(i)
  • Далее после окончания 1-ой итерации происходит увеличение (i) на единицу при помощи выполнения инструкции i++
    (i) становится теперь 1
  • Перед 2-ой итерацией мы опять смотрим на условие i < 5
    Для того, чтобы выполнилась следующая итерация условие должно быть правдиво(true), переменная i должна быть меньше 5. Поскольку (i) после первой итерации получает значение 1, и это меньше 5, то выполняется следующая итерация. Выполнение последующих итераций будет происходить до тех пор, пока условие не станет ложным, и происходит выход из цикла. В нашем случае переменная i должна быть меньше 5.
Папка 01. Для перебора массивов не нужен for

Цикл for не рекоменжуется использовать для массивов

Цикл for можно использовать для массивов, но не рекомендуется. Для этого существуют функции высшего порядка массивов. Функции for each и map, являются по сути методами прототипа массивов. И они доступны для каждого массива. С помощью таких функций нужно перебирать элементы массивов.

Тем не менее разберем пример использования цикла for для массивов.

                    
                        // цикл for для массивов

                        const myArray = ['first', 'second', 'third'];
                        
                        for (let i = 0; i < myArray.length; i++) {
                            console.log(myArray[i])
                        }
  • 1) переменной i присвоено значение 0
    let i = 0;
  • 2) сравнение значения переменной i с длиной массива
    i < myArray.length;
  • 3) увеличение значения переменнной i на единицу
    i++
  • 4) Вывод в консоль браузера
    console.log(myArray[i])

В выводе в консоли браузера должно быть следующее:
first
second
third
3 элемента этого массива.

Когда переменная i станет равна 3, мы выдем из цикла. Потому, что условие сравнения значения переменной i с длиной массива станет ложным. Длина массива - 3

Папка 02. Для работы с массивами нужно использовать метод foreach 

                    
                        const myArray = ['first', 'second', 'third'];

                        myArray.forEach((element, index) => {
                            console.log(element, index)
                        })

В цикле for предыдущего примера мы получали доступ к элементам массива в выводе консоли браузера, используя квадратные скобки 

                    
                        console.log(myArray[i])

[i] - представлял по сути индекс определенного элемента в массиве. Индекс в массиве начинает отсчитываться от 0

Используя foreach, во-первых, вы можете получать доступ непосредственно к элементам массива, а кроме того, если хотите можете также получить доступ к индексу определенного элемента.

Метод foreach вызывается с помощью точечной записи. После названия перемнной ставится точка и пишется метод forEach

                        
                            myArray.forEach

В этом методе forEach вы долджны указать callback функцию, которая вызывается столько раз, сколько элементов в массиве. В данном примере эта callback функция вызывется трижды.

В этой callback функции мы вызываем 2 параметра: (element, index), где element будет представлять каждый элемент массива, index будет индексом каждого элемента (отсчет индекса идет от 0). Второй параметр(element) в этой callback функции опционален. Можно оставить только один(element), если вам не нужен индекс.

В данном случае в выводе консоли браузера мы вызываем доступ к элементам и к индексу элементов массива. И получаем их. И будем видеть вот это:
first 0
second 1
third 2

callback функция, которая передается в качестве аргумента метода forEach, будет вызываться столько раз, сколько элементов в исходном массиве. foreach создан для того, чтобы перебирать элементы маассива. То есть foreach возвращает undefined , потому нет смысла присваивать результат вызова метода foreach какой-то переменной.
