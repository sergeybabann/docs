Знакомство с NodeJS

Выполнение файла с кодом JavaScript с помощью NodeJS в терминале

Для того, чтобы выполнить определенный файл с кодом JavaScript нужно использовать интерпретатор Node. В терминале набираем команду, которая запустит файл с кодом JavaScript: 

                    
                        node index.js

Выполнять код JavaScript можно также внутри интерактивного интерпретатора NODE REPL. REPL - это аббревиатура.

  • R - READ (Прочитать код введенный пользователем)
  • E - EVALUATE (Анализ и интепретация кода)
  • P - PRINT (Вывести результат в терминал)
  • L - LOOP (Начать сначала. При этом переменные, созданные ранее, сохраняются)

Запуск интерактивного интерпретатора NodeJS в терминале

Для того, чтобы запустить интерактивный интерпретатор NodeJS в терминале, нужно запустить команду: 

                    
                        node

Выход из интерактивного интепретатора

Для выхода из REPL нужно запустить команду: 

                    
                        .exit

Получение помощи в NodeJS

С помощью команды .help можно получить помощь NODE REPL. Вводим в терминале .help и получаем список доступных команд c описанием:

  • .break - Прервать выполнение действия определенного кода, если это выполнение занимает много времени.
  • .clear - Выполняет тоже самое, что и команда .break
  • .editor - Запустить редактор, в котором можно написать блок кода JavaScript.
  • .exit - Выйти из текущей сессии REPL.
  • .help - Получить помощь.
  • .load - Загрузить определенный файл JavaScript непосредственно внутри сессии REPL.
  • .save - Сохранение всех команд, введенных в интерактивном интерпретаторе в файл.
Практика - Выполнение кода в Node.js

Создадим файл index.js с кодом 

                    
                        console.log('Hello Node.js!')

Запустим этот файл на выполнение в терминале с помощью интерпретатроа NodeJS. Набираем команду: 

                    
                        node index.js

И видим в выводе текст, который в скобках - Hello Node.js! Это результат выполнения файла index.js с помощью интерпретатора NodeJS. Программа была выполнена и завершила свою работу. Также можно не указывать расширение файла, а просто написать его название.

Увеличим код, и запустим его в терминале: 

                    
                        console.log('Hello NodeJS!')

                        const myName = 'Bogdan'
                        console.log(myName)

А теперь выполним этот код в файле при помощи расширения code runner.

Таким образом можно выполнть код в файлах javascript 2-мя способми:
1) Выполнять код в терминале командой: node имя файла, например - node index.js
2) При помощи расширения Code Runner

Практика - Использование Node REPL

Запустим в терминале команду node. Мы попадаем в интерактивный интерпретатор Node REPL. Запустим следующий код: 

                    
                        const myHobbies = ['swimming']

В ответе мы получаем indefined. Это то, что возвращает интерпретатор при создании новой переменной, и при присвоении ей значения. Это не выражение, это(стэйтмент) инструкция в JavaScript. А теперь давайте обратимся к переменной myHobbies - и мы просто набираем ее имя. И мы видим массив с 1 элементом.

Добавим 1 элемент в массив: 

                    
                        myHobbies.push('hiking')

В ответе я вижу новую длину массива - 2. Это то, что возвращает метод push. Это у нас выражение. Вызов любой функции в JavaScript - это выражение. Посмотрим теперь на значение переменной myHobbies. Просто наберем ее имя. Массив из 2 элементов.

И теперь давайте посмотрим на значение переменной global. Набираем global. Это большой объект, в котором есть ряд свойств. И теперь давайте посморим на то, как можно набрать код в редакторе. Команда .editor

Мы попали в редактор и здесь набираем код, просто набор без enter: 

                    
    const myName = 'Bogdan'
    console.log(myName)
    const myArray = [1, 5, 10]
    console.log(myArray)
    console.log(myHobbies) // переменная набранная ранее остается в памяти этой сессии

Теперь нажимаем хоткей ctrl+d, и весь код в этом редакторе был только что выполнен, и мы видим результат выполнения - 3 вывода в терминал.

Сохраним все команды, которые мы вводили в этой сессии в файл. Команда .save имя файла.

Набирам команду .save commands.txt , и выходим из этой сессии - команда .exit

Теперь посмотрим в текущей директории список файлов и папок. Команда - ls, и мы видим, что в нашей рабочей директории есть файлы. Посмотрим содержимое файла commands.txt - команда cat commands.txt

И я вижу все команды, которые вводил в интерактивной сессии Node REPL.

Ввводим переменную myHobbies и видим что она уже не существует, потому что это новая сессия. Доступ к переменной есть, пока мы не вышли из сессии. Выходим из текущей сессии и удалим наш файл commands.txt - команда rm commands.txt
