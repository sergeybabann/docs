Это то, что отличает React от других библиотек и фреймворков для создания одностраничных приложений.

09. Компоненты

Любое приложение React состоит из компонентов.

Характеристики компонентов.

  • Компоненты можно использовать многократно
  • Компоненты формируют четкую иерархическую структуру. Есть родительские компоненты, которые включают в себя дочерние компоненты. А также есть один корневой компонент в любом React приложении.
  • Видом компонентов можно управлять извне с помощью свойств, которые передаются от родительского компонента дочернему компоненту. Это можно сказать как аргументы при вызове функции.
  • Компоненты могут иметь свое собственное состояние, и это состяние может изменяться в процессе жизненного цикла приложения, а также жизненного цикла самого компонента.
  • Каждый компонент отвечает за отдельную логическую часть интерфейса.
10. Иерархия и переиспользование компонентов

Корневой компонент

Корневой компонент включает в себя другие компоненты

Видимые компоненты

Невидимые компоненты необходимы для того, чтобы создать иерархию компонентов. Видимые компоненты содержат определенную часть интерфейса для пользователя.

Переиспользование компонентов.

С помощью компонентов можно создавать другие страницы, с другим расположением компонентов на странице. С помощью одних и тех же компонентов можно создавать разные видимые части интерфейса.

Иерархия компонентов.

У каждого компонента должен быть родительский компонент, за исключением корневого компонента.

12. Функциональные и классовые компоненты

Функциональные компоненты

В современных приложениях React любой компонент можно создать с помощью функции JavaScript , и такие компоненты называются функциональными компонентами. И здесь можно использовать традиционные функции, которые объявляются используя ключевое слово function : 

                    
                        function Helloworld() {
                            return <h1>Hello World</h1>
                        }

, либо же стрелочные функции : 

                    
                        const Helloworld = () => {
                            return <h1>Hello World</h1>
                        }

Стрелочная функция начинается с круглых скобок, и в данном примере она присваивается переменной Helloworld
То есть можно использовать как традиционные функции, так и стрелочные функции. Преимущество использования стрелочных функций в том, что переменная, которая объявлена в JavaScript, используя const , не может иметь другое значение. То есть переменной Helloworld невозможно присвоить другое значение.

То, что вы видите после ключевого слова return , это синтаксис JSX

Классовые компоненты 

                    
                        class Helloworld extends Component {
                            render() {
                                return <h1>Hello World</h1>
                            }
                        }

Также компоненты в React можно создавать с помощью классов JavaScript. При этом такие классы обязательно должны иметь метод render , который определяет то, что подлежит рендерингу. И в данном примере из метода render также возвращается JSX , то есть то, что React приложение должно отобразить в видимой части интерфейса пользователя.

В современных приложениях React большинство компонентов - это Функциональные компоненты, а Классовые компоненты почти никто не использует.

13. JSX - JavaScript Syntax Extension

JSX — это специальный синтаксис, с помощью которого можно объединять HTML, CSS и JavaScript. Тем самым формировать полноценные компоненты React

Папка 01. Рассмотрим структуру файла с примером JSX

Структура этого примера похожа на HTML:

  • Открывающий тег под названием Card , и у этого тега есть 2 атрибута: style и className, и у этого тега есть также закрывающий тег Card
  • Тег Card является родительским элементом, и у него есть дочерний элемент - Card.Img , у которого также есть свои атрибуты, такие как: variant, style, src и т.д. То есть дочерний элемент с определенными собственными атрибутами.
  • При этом Card и Card.Img похожи на HTML теги, но таковыми не являются, это компоненты React
  • То есть в приложении, в котором вы найдете такой JSX должны быть компоненты с названием Card и Card.Img
  • Также в этом примере есть синтаксис, который похож на CSS, в нем есть атрибут style, и его значение - это объект JavaScript на самом деле. И в этом объекте JavaScript есть свойство backgroundColor , а значение этого свойства - это JavaScript
    И в этом выражении JavaScript мы используем переменные initialColor и opacity
  • Также в этом примере есть другая часть JavaScript , и здесь видна Стрелочная функция JavaScript:
    () => setImageLoaded(true)

Синтаксис JSX похож на HTML и CSS , но в нем все реализовано с помощью JavaScript, React - это JavaScript, виртуальный DOM - это JavaScript объект. Но с помощью виртуального DOM , который состоит вот из таких компонентов, React и может взаимодействовать с реальным DOM , который состоит из HTML элементов.

Card и Card.Image являются компонентами React , а не HTML элементами. Внутри тега Card вы видите JavaScript объект style, с помощью которого описываются стили CSS

JSX - Это всего лишь синтаксическая надстройка JavaScript.

JSX код из этого примера конвертируется в вызов функции createElement(файл - function_call.js), которая является методом объекта React. И как видно метод createElement вызывается в данном случае дважды, потому что у нас 2 элемента JSX. И первым аргументом в вызове каждого метода вы видите название компонента React: Card и Card.Img

Также в первом вызове метода createElement вторым аргументом здесь отображен JavaScript объект, который размещен между фигурными скобками, которые открываются сразу после 1-ого аргумента Card и закрываются перед следующим вторым вызовом. И в этом JavaScript объекте есть 2 свойства: style и className
Эти свойства передаются в компонент Card, и эти свойства собраны в одном JavaScript объекте.

14. Встроенные и пользовательские компоненты

Пользовательские и встроенные компоненты.

React поставляется с определенным набором уже готовых к использованию компонентов. И в данном примере, там где вы видите <div>, либо <h1>, либо <p> , вы как раз и используете встроенные в React компоненты. Это не HTML теги, это React компоненты. А внутри React создана уже логика касательно того, как создавать соответствующие HTML теги непосредственно в реальном DOM. JSX конвертируется в JavaScript код.

А вот Footer в данном примере является компонентом, который должны создать вы, как пользователь React. То есть Footer - это компонент, который должен быть создан в вашем React приложении. И React отличает встроенные компоненты от пользовательских по первой букве, которая указана в открывающем теге JSX. Заглавная буква - пользовательский компонент. Маленькая буква - встроенный компонент.

Потому в React есть правило - Все компоненты, которые вы создаете в ваших React приложениях должны быть написаны с большой буквы. Именно таким образом React понимает, что в том или ином месте JSX вы хотите использовать ваш компонент, а не встроенный компонент.

15. Корневой элемент в JSX

В React JSX должен иметь один корневой элемент.

JSX может возвращаться из функции с помощью ключевого слова return , если вы создаете функциональный компонент. Либо же из метода render , если вы создаете классовый компонент.

Примеры валидного и не валидного JSX кода. В примере валидного JSX вы видите, что родительским компонентом для всего кода является тег div, который является корневвым компонентом в этом JSX коде. В примере невалидного кода компоненты h1 и p находятся рядом и у них нет одного общего родительского элемента. Такой код является невалидным в React. А почему? Ответ очень простой:

Потому что, если вы сконвертируете вот такой невалидный JSX код в JavaScript , то в результате вы получете 2 выражения JavaScript. И такой код является невалидным кодом JavaScript. Именно поэтому вы должны всегда иметь один корневой элемент в любом JSX коде.

16. Свойства и состояние компонентов

Свойства (PROPS) и состояние (STATE)

Свойста передаются дочерним компонентам от родительских, и они позволяют влиять на внешний вид того или иного компонента, а также на его поведение. А вот состояние может меняться внутри конкретного компонента. И состояние меняется на протяжении жизненного цикла того или иного компонента. Разберем подробнее...

Есть компонент React, снаружи к нему приходят свойства от его родительского компонента. Родительский компонент может быть только один. Но вы можете использовать один и тот же компонент React в разных местах. Тем самым у каждого такого экземпляра компонентов будут разные родительские компоненты.

То есть снаружи React компонент получает свойства, далее внутри компонента React может быть состояние. Но некоторые компоненты React могут не иметь такого состояния. Такие компоненты называются STATES - без состояния. И вот состояние в компоненте React уже может изменяться на протяжении жизненного цикла компонента.

Кроме того нужно еще раз отметить, что изменение состояния конкретного компонента не влияет на другие вышестоящие по иерархии компоненты. В том числе на родительский компонент, который передает свойства этому компоненту. Также каждый компонент React может передавать свойства дочерним компонентам, если таковые имеются. При этом у одного компонента React может быть несколько дочерних компонентов.

17. Родительские и дочерние компоненты

Принцип иерархии взаимодействия компонентов друг с другом при передаче значений свойств от одного компонента к другому компоненту.

У каждого компонента в React, за исключением корневого компонента, есть родитель(parent)- родительский компонент. И у одного родительского компонента может быть несколько дочерних компонентов. То есть соотношение один ко многим. При этом один дочерний компонент может иметь только один родительский компонент. Исходя из этого, в React существует еще один важный принцип, а именно: Свойства компоненту передаются от родительского компонента. Это очень важный принцип. При этом передача свойств возможна от родителя дочерних компонентов, но обратно от дочернего компонента родительскому компоненту свойства передать нельзя. То есть передача свойств всегда односторонняя. Начиная от корневого компонента к его дочернему компоненту, этот дочерний компонент передает своим дочерним компонентам и т.д. Так путешествует свойство по компонентам в React.

Все компоненты имеют четкую иерархию, и из-за этого нельзя передавать свойства от дочернего компонента к родительскому компоненту. Также нельзя передавать свойства от вышестоящего в иерархии компонента напрямую нижестоящему, минуя его родительские компоненты. Это нарушает иерархический принцип иерархии компонентов в React приложении. Поэтому передаваемые свойства необходимо пробрасывать через все промежуточные компоненты, которые находятся между этими двумя компонентами.

18. Принципы изменения свойств и состояния

Компонент не должен изменять собственные свойства

Собственные свойства необходимо рассматривать как вводные параметры для определенного компонента. И эти параметры изменять не следует. Технически их можно изменять, но крайне не рекомендуется. Потому что свойства конкретного компонента определяются родительским компонентом. И именно родительский компонент передает определенные свойства своим дочерним компонентам. Но при этом компонент может изменять собственное состояние, если таковое имеется у определенного компонента.

Компонент не может изменять состояние других компонентов, включая родительских и дочерних компонентов.

То есть существует такое правило - в React состояние должно меняться в том компоненте, который является собственником этого состояния, компонентом, в котором это состояние было создано. При этом можно влиять на изменение состояния других компонентов. Подробнее...

Допустим у компонента есть определенное состояние. У нижестоящего через одного компонента изменение его состояния может повлиять на состояние компонента его прородителя. Но непосредственно изменение состояния происходит именно в компоненте прородителе. Таким образом любой из компонентов ниже по иерархии может отправить запрос на изменение состояния вышестоящего компонента.

19. Ререндеринг компонентов

Компонент подлежит ререндерингу при изменении свойств или состояния

Каждый раз когда меняются свойства либо состояние компонента, React выполняет ререндеринг определенного компонента. То есть происходит изменение в виртуальном DOM, текущая версия сравнивается с предыдущей версией виртуального DOM, находятся отличия, и потом если необходимо изменить уже реальный DOM , то React вносит такие изменения. При этом сам React этим не занимается, это делает другая библиотека.

20. Резюме по ключевым понятиям в React
  • Компоненты
  • JSX
  • Свойства
  • Состояние

Компоненты - это строительные блоки любого React приложения. Компоненты выстраиваются в четкую иерархическую структуру. У каждого компонента кроме корневого должен быть один родительский компонент, а также у любого компонента может быть много дочерних компонентов. Компоненты можно при этом переиспользовать.

JSX - это синтаксическая надстройка над JavaScript. С помощью JSX, используя JavaScript, а также синтаксис похожий на CSS и похожий на HTML, можно описать любой компонент React. Но в конечном итоге JSX конвертируется в JavaScript.

Свойства - это вводная информация для любого компонента, и свойства передаются от родительского компонента к дочернему компоненту. При чем свойства внутри компонента менять крайне не рекомендуется. Они должны меняться только в родительском компоненте. Но если нужна динамика в компоненте, то есть динамическое поведение компонента непосредственно, то необходимо использовать состояние(STATE). При этом тот компонент, в котором состояние и создано, должен быть единственным компонентом, который имеет право изменять это состояние. Кроме этого части свойств и состояния можно передавать другим дочерним компонентам в виде свойств.

21. React Hooks

Ключевое понятие в React - React Hooks

Хуки React появились в версии 16.8 , и с помощью React Hooks можно управлять состоянием в функциональных компонентах. До появления React Hooks состоянием можно было управлять только в классовых компонентах React. Функциональные компоненты можно было использовать только для STATLES компонентов, компонентов без собственного состояния. Сейчас благодаря таким React Hooks, как useState и useEffect, можно легко управлять состоянием в функциональных компонентах. И при этом необходимость в использовании классовых компонентов отпадает.
