53. Почему в JSX нужно использовать className вместо class

В нашем JSX коде нам необходимо внести некоторые изменения. При добавлении JSX кода вместо атрибута class для компонента React(потому, что div - это компонент, button - это компонент, хотя они и встроенные) мы должны использовать className. Это все потому, что JSX это надстройка над JavaScript. А в JavaScript в DOM API классы добавляются элементам с помощью className.

Причина, почему используется className в том, что мы работаем здесь с JavaScript, а не с DOM API, как это делается в Vanilla.js. Потому вместо class мы должны использовать className, если вы хотите указать какие-то классы для определенных компонентов.

56. Запуск проекта React с помощью Live Server

Давайте попробуем запустить этот проект 05_react_jsx_green_button с помощью LiveServer

Для того, чтобы установить пакет npm live-server в проект локально, нужно запустить определенную команду в терминале. Перейдем по ссылке и посмотрим доступные команды. На официальном сайте npm есть команда, позволяющая установить пакет в проект локально. Вот эта команда: npm i live-server

Существует возможность запуска определенного npm пакета однократно с помощью npx. И этим мы сейчас и воспользуемся.

Мы должны запустить определенную команду в терминале, находясь выше на одну директорию от корневого каталога проекта, который собираемся запустить при помощи npx. В данном случае из директории react-projects_user. Запускаем необходимую команду: npx live-server --open=05_react_jsx_green_button И проект запустился в веб-браузере.

Откроем console в веб-браузере, и здесьесть пару ошибок, а также предупреждение.

Это предупреждение мы уже видели ранее:

You are using the in-browser Babel transformer. Be sure to precompile your scripts for production

Мы все еще используем Babel для трансформации JSX в JavaScript непосредственно в веб-браузере, потому мы это предупреждение можем проигнорировать.

Далее есть ошибка:

Failed to load resource: the server responded with a status of 404 (Not Found)

И эта ошибка касается того, что любой веб-браузер при подключении к веб-странице запрашивает favicon. Это иконка, которая показана возле заголовка страницы в самом верху над поиском. Но так как в нашем проекте иконки нет, то веб-сервер отвечает: 404 (Not Found). То есть такая иконка не найдена. Поэтому эту ошибку мы можем также проигнорировать.

Кроме этого мы видим еще одну ошибку, которая помечена как Warning.

57. Использование JavaScript в коде JSX

Теперь давайте подумаем как при нажатии на эту кнопку можем изменять вот этот текст, который находится между открывающим JSX тегом button и закрывающим JSX тегом button.

В файле script.js добавим новую переменную const buttonText, и значение для этой переменной будет текст Click me.

Так как мы написали только что JavaScript код, то используем здесь одинарные кавычки. А для значений свойств, которые передаются в компонент, мы будем использовать двойные кавычки, также это происходит для обозначения атрибутов html тегов.

Итак, мы создали переменную buttonText. Как же мы теперь можем получить доступ к значению этой переменной в коде JSX? Очень просто - с помощью пары фигурных скобок. Каждый раз, когда вы хотите использовать JavaScript в JSX, вы должны добавлять пару фигурных скобок. И внутри этих скобок вы можете написать абсолютно любое выражение JavaScript. А выражение всегда возвращает определенное значение, потому мы здесь можем написать просто: buttonText

Сохраняем код, и проверим что-же изменилось в веб-браузере?

Итак, ничего не изменилось. Мы все еще видим кнопку с текстом Click me. Во вкладке console мы видим те же ошибки что и ранее, и мы можем пока их проигнорировать. Перейдем в elements, чтобы убедиться, что текст для кнопки корректно появился в html разметке между тегами button непосредственно в реальном DOM, и мы это видим.

Изменим значение переменной buttonText в JavaScript, например напишем Click me please, вместо Click me, и посмотрим результат. И текст на кнопке изменился.

То есть это работает корректно. Мы теперь можем получать доступ к значениям переменных JavaScript, например, используя пару фигурных скобок, непосредственно в JSX.

60. Перенос кода JSX в функциональный компонент React

Так вот для того, чтобы в React это все начало работать корректно, и чтобы на кнопке мы увидели корректный текст, который мы меняем при нажатии на кнопку, часть кода нашего скрипта мы должны перенести в одну функцию. А именно в функциональный компонент, который будет возвращать наш JSX. Также нам будет необходимо довавить состояние этого компонента.

Итак приступим по порядку. Сначала создадим функциональный компонент, в который мы и перенесем необходимый код. Вместо присваивания JSX кода переменной App, мы здесь присвоим ей стрелочную функцию. Так как функциональный компонент должен возвращать JSX, мы здесь пишем return и вставляем обратно наш JSX.

Теперь у нас есть фунциональный компонент, который возвращает JSX код.

И теперь для того, чтобы мы корректно могли отобразить этот компонент в пользовательском интерфейсе, вместо названия переменной в вызове метода render, мы должны также указать JSX код. Открывающий тег, далее название компонента, и после названия компонента мы должны добавить пробел и слэш для того, чтобы сделать этот JSX тег самозакрывающимся. Вот так:

Напоминаю о том, что все пользовательские компоненты должны начинаться с заглаавной буквы. Именно так React понимает, что вы хотите добавить пользовательский компонент, а не встроеный. Такой, например, как div , либо button. То есть здесь мы хотим отобразить в интерфейсе наш собственный компонент. А создали мы наш компонент с помощью синтаксиса функциональных компонентов.

Итак, кроме этого мы должны в эту функцию const App перенести переменные.

Давайте посмотрим как работает приложение сейчас.

Нажимаем на кнопку, и снова текст на кнопке не меняется. А это происходит из-за того, что значение переменной buttonText мы изменяем внутри этой функции. И React никак не может отследить то, что происходит внутри функции.

В самом верху добавим еще один console.log и в нем напишем просто (Called). Это будет означать, что функция была вызвана. Сохраним изменения и запустим приложение.

Заходим в console и видим, что функция была вызвана. Нажимаем на кнопку, видно вывод Hello from React, но не видно, что функция еще раз вызывалась. Это означает, что функция App была вызвана однократно и вызвал ее React, для того чтобы получить JSX код, который мы из этой функции вернули. Далее обработать его в виртуальном DOM и перенести в реальный DOM. И все! На этом работа React закончилась.

То, что мы изменяем значение переменной buttonText внутри функции при нажатии на кнопку, никак не влияет на перерисовку этого компонента в интерфейсе. Потому что для того, чтобы перерисовать компонент в интерфейсе, необходимо вызвать еще раз эту функцию. Так работает React.
