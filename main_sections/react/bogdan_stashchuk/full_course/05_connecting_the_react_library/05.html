33. Создание h1 элемента с помощью React

Теперь при помощи объекта React мы можем создавать новые компоненты. Для этого используется метод createElement. И так создадим новый элемент.

Вызываем createElement и ему передаем название компонента React(название компонента находится в скобках), который мы хотим создать. Пишем 'h1' , вторым аргументом вы здесь можете передать опции для элемента, и это на самом деле набор свойств для определенного комопонента React. Сейчас мы не будем указывать свойства для элемента h1, потому можно написать здесь просто null . Но мы хотим указать текст, который будет отображен внутри тега h1, и напишем текст 'Hello from React'

То есть мы вызываем метод createElement и ему передаем 3 аргумента:

  • h1 - это название компонента, который мы хотим создать
  • второй аргумент у нас null, потому что мы не передаем объект со свойствами для этого компонента h1
  • 3-ий аргумент - это то, что будет находиться внутри тега h1, в данном примере у нас идет текст.

И так - Мы вызвали метод createElement объекта React, и мы создали новый элемент React и присвоили результат вызовы этого метода переменной helloWorldHeading

Теперь давайте просто выведем в консоль веб-браузера этот объект helloWorldHeading, а это будет объект. И в консоли мы видим вот такой вот объект со свойствами type и props. Можно его развернуть и здесь мы увидим тип этого объекта как h1 , полностью - type: "h1", а также мы видим свойства под названием props

Значением этого свойства props является объект с одним свойством children, и значение его Hello from React, это 3-ий аргумент в вызове метода createElement

И так мы видим, что результатом вызова метода createElement является объект. Это то, что возвращает этот метод. И мы в консоли веб-браузера видим структуру этого объекта. Для нас важен тип этого элемента React, а также то, что будет находиться внутри этого элемента. А внутри в данном примере находится текст Hello from React

И так мы получили вот такой вот объект в результате вызова метода createElement, но на самой странице index.html ничего не изменилось, она пустая. Перейдем в инструментах разработчика веб-браузера на вкладку elements и развернем body, мы здесь увидим только скрипты, никаких других тегов HTML здесь нет. И мы не видим тега h1 после его создания с помощью вызова метода createElement

То, что мы не видим в структуре HTML документа тега h1 это вполне логично, мы нигде его не добавляем в нашу страницу. Получается что с помощью React мы смогли создать определенный элемент React, и мы убедились в том, что это просто объект. И теперь нам необходимо каким-то образом показать его в веб-браузере.

34. Подключение библиотеки react-dom

Еще раз скажем то, что React - это библиотека для пользовательских интерфейсов. То есть с помощью нее создаются компоненты React. А вот показать эти компоненты сам React не может. Для этого необходимо использовать другую библиотеку, которая для веб-браузеров называется React-Dom. Для того, чтобы отобразить этот элемент React в нашей странице, мы должны подключить библиотеку React-Dom.

Подключение библиотеки к HTML странице происходит так же, как и подключение библиотеки React, с помощью внешней ссылки.

Перейдем в веб-браузер. И теперь в веб-браузере у вас будет доступна переменная ReactDOM. Во вкладке console напишем ReactDOM и нажимаем enter
И это объект, также как и переменная React, и в этом объекте доступны такие методы как, createRoot, flushSync и т.д.

Как раз с помощью этого объекта ReactDOM и происходит рендеринг Реакт компонент на HTML странице. То есть выполняется процесс взаимодействия с реальным DOM. А вот за виртуальный DOM отвечает React, а за взаимодействие с реальным DOM отвечает React-DOM.

Для этого в HTML документе необходимо место, куда React-DOM поместит определенный компонент React. Добавим новый div c id сразу после body. 37. Проверка успешного добавления React компонента в DOM

Запустим наш html документ, и мы видим текст Hello from React. Это означает, что html тег h1 был успешно помещен на веб-страницу.

Перейдем на вкладку elements и проанализируем то, что видно в теле html документа.

Здесь мы видим div с id="app", и внутри него появился дочерний элмент html, тег h1 с текстом Hello from React. То есть мы успешно с помощью React-DOM смонтировали компонент React, который находится в переменной helloWorldHeading, непосредственно в реальный DOM. Для этого мы нашли элемент по id="app", создали корневой элемент с помощью React-DOM, и после этого выполнили render компонента React внутри этого корневого элемента.

38. Проект по добавлению h1 с помощью Vanilla JS

Давайте здесь выполним ту же самую операцию, то есть добавим тег html h1 внутри div, но с помощью vanilla javascript, без React-DOM. Сделаем это в отдельном примере, создадим новый файл index.html и скопируем в него все содержимое нашего html документа.

В директории, где хранятся все наши проекты React, создадим новую директорию с названием 03_react_vs_vanilla-js, и откроем этот проект.

Переименуем div id="app" на div id="react-app", также это нужно сделать в скрипте в том месте, где мы находим html элемент по id. То есть в константе container в вызове метода getElementById объекта document. Добавим еще один div с id , где название id будет vanilla-app. Теперь наша задача добавить h1 внутри этого div, но без использования React, а с помощью vanilla.js

И сделаем мы это в отдельном скрипте javascript. Добавляем еще один скрипт сразу после существующего скрипта, и здесь создадим новый элемент html.

Объявим переменную с названием heading и присвоим ей результат вызова метода createElement объекта document. Таким образом можно создавать любые html элементы. В скобках мы пишем название html элемента, который мы хотим создать. И это в данном случае h1. Далее нам необходимо добавить текст внутри этого html элемента. Делаем это при помощи метода textContent объекта heading.

И так мы создали html элемент h1 с текстом, и теперь его необходимо поместить внутри элемента div. Для этого мы можем воспользоваться методом appendChild. После того как мы нашли html элемент по его id с помощью метода getElementById объекта document, мы обращаемся через точку к методу appendChild. Методу appendChild передаем элемент heading , который мы хотим добавить внутри элемента vanilla-app.

Вот и все! Можем запустить наш пример. И мы должны увидеть здесь еще одну строчку текста, которая создается при помощи нашего скрипта без применения React, а чистого Vanilla.js. И эта строчка - Hello from Vanilla JS

39. Сравнения результатов добавления h1 с React и без него

Откроем наш html документ и в скрипте который работает с помощью React переименуем переменную helloWorldHeading на reactHeading, также мы должны это сделать в вызове метода render объекта root.

В скрипте, где используется Vanilla.js переименуем переменную heading на vanillaJsHeading, также делаем это во всех местах где это необходимо. Пропишем в этом скрипте команду console.log, где мы выведем в консоль vanillaJsHeading. Также выведем в консоль переменную reactHeading в скрипте React. Сохраним изменения и перейдем на вкладку console в веб-браузере.

Здесь я вижу объект, как результат вывода в консоль переменной reactHeading, то есть тот объект который вернул метод createElement. Развернем структуру объекта и видим,что у этого объекта есть type: "h1", а также свойства(props). За пределами описания структуры объекта мы видим элемент h1 с текстом Hello from Vanilla JS. Это тот текст с которым мы работаем в скрипте.

Назовем отличие этих двух элементов.

В случае, где скрипт c помощью React, мы видим компонент React. И для того чтобы показать его на html странице нам понадобилась библиотека React-DOM, которая сконвертировала получается этот объект в html. Далее выполнился рендеринг компонента React непосредственно в реальном DOM.

В скрипте Vanilla.js мы создали html элемент напрямую. И конечно же такой элемент мы можем добавить на html страницу непосредственно, с помощью вызова метода appendChild, если мы хотим добавить такой элемент внутри другого родительского элемента.

И так : в случае со скриптом React в консоли виден объект JavaScript , а в случае со скриптом Vanilla.js виден непосредственно элемент html.
