1 Теория. Что такое React.

1.01 Что такое React.

React - это библиотека JavaScript, которая позволяет строить какие-либо веб-приложения и конкретно то, что запускается у вас в веб-браузере. React позволяет строить сложные и реактивные пользовательские интерфейсы. Все элементы, которые есть в React они делятся на компоненты.

1.03 Что такое компоненты

Закомментируем сам React, напишем какой-нибудь код. Создаем div с классом car , здесь мы создадим html код, который будет описывать машину. Создадим инлайновые стили на странице.

Создадим еще одну машину Audi A8. Как мы это сделаем? Просто сделаем копипаст в разметке и изменим данные. У нас появился другой элемент, который описывает другую машину. Структура у этих 2-ух элементов абсолютно одинаковая. Есть класс car, заголовок h3, параграф. То есть структура данных компонентов одинаковая, но при этом содержимое у них разное.

Мы создали 2-ой компонент при помощи дублирования html кода, и в принципе этот подход нормальный, но при этом, если мы захотим создать здесь еще одну машину, то у нас количество кода будет заметно увеличиваться. Создадим еще одну машину при помощи копипаст html.

По сути мы создали компоненты. Каждый из этих div с классом car содержит в себе определенную структуру, и может уже называться компонентом. React построен на данных компонентах. То есть он предлагает создавать некоторую функцию, которая будет возвращать структуру определенного компонента, и после этого мы уже сможем настраивать каждый из них.

Давайте посмотрим как мы можем это сделать. У нас уже есть настроенный React.

Создадим новую функцию, которая будет называться Car. Это обычная функция js, которая будет делать следующее: Она будет возвращать ту структуру компонента, которуая будет описана в этой функции.

Копипастим html какой-нибудь машины в return, не забываем про className. И попробуем что-нибудь сделать с данной функцией. На всякий случай не будем удалять закомментированный код React, но напишем все с нуля.

Наша функция Car является по сути простым React компонентом. То есть React компонент - это обычно какая-нибудь функция. Но теперь нам нужно каким-то образом передать ее в шаблон, и для этого как раз у нас есть div="root".

Для того, чтобы занести в html данный компонент мы должны сделать следующее: Мы можем обратиться к библиотеке React-DOM, которая ответственна за то, чтобы заносить какие-либо данные в html, и вызываем метод render, с помощью которого как раз таки и происходит инъекция какого-либо React компонента в html.

1-ым параметром мы должны указать здесь, какой компонент мы хотим занести внутрь html.

2-ым параметром мы должны указать тот корневой блок, куда мы хотим занести всю эту структуру. То есть div="root", поэтому нам достаточно просто его забрать из html. В JavaScript это делается довольно просто - либо с помощью стандартной функции document.getElementById , либо вот так: document.querySelector("#root"), например указываем в скобках в качестве параметра элемент с id="root".

У нас есть функция Car, и мы должны ее как-то положить. Есть ключевое слово, которое описывает название данной функции, и по сути все сводится к тому, что мы данное название должны превратить в новый html тег. И React поймет, что это новый компонент. И по сути мы пишем новый тег, и внутри него пишем название Car. И указываем это в качестве 1-ого параметра.
